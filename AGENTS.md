## 项目概述
- **名称**: 银行流水PDF转Excel工作流（视觉OCR版本）
- **功能**: 使用视觉模型识别PDF表格，提取银行流水数据，生成统一格式的Excel文件，自动验证余额并标记问题行

### 节点清单
| 节点名 | 文件位置 | 类型 | 功能描述 | 分支逻辑 | 配置文件 |
|-------|---------|------|---------|---------|---------|
| pdf_to_images | `nodes/pdf_to_images_node.py` | task | 将PDF文件转换为图片（300 DPI），上传到对象存储 | - | - |
| bank_identify | `nodes/bank_identify_node.py` | agent | 使用视觉模型从图片中识别银行类型、账户户名和账号 | - | `config/bank_identify_llm_cfg.json` |
| ocr_recognition | `nodes/ocr_recognition_node.py` | task | 使用视觉模型识别图片中的表格，逐行提取数据 | - | - |
| transaction_extract | `nodes/transaction_extract_node.py` | agent | 从OCR结果中提取结构化的交易记录数据 | - | `config/transaction_extract_llm_cfg.json` |
| data_validation | `nodes/data_validation_node.py` | agent | 校验交易记录数据格式，验证余额正确性，标记问题数据 | - | `config/data_validation_llm_cfg.json` |
| excel_export | `nodes/excel_export_node.py` | task | 将校验后的交易记录导出为统一格式的Excel文件，问题行标黄 | - | - |

**类型说明**: task(普通节点) / agent(大模型节点) / condition(条件分支) / looparray(列表循环) / loopcond(条件循环)

## 技能使用
- 节点`pdf_to_images`使用pdf2image库将PDF转换为图片，使用Storage技能上传图片
- 节点`bank_identify`使用视觉模型（doubao-seed-1-6-vision-250815）识别银行信息
- 节点`ocr_recognition`使用视觉模型识别表格，逐行提取数据
- 节点`transaction_extract`使用大语言模型从OCR结果提取结构化数据
- 节点`data_validation`使用大语言模型校验和修正数据，验证余额
- 节点`excel_export`使用openpyxl生成Excel文件，问题行标黄

## 核心功能
1. **PDF转图片**：将PDF文件转换为高清晰度图片（300 DPI），支持多页PDF
2. **视觉识别银行信息**：使用视觉模型从图片中识别银行名称、账户户名、账号
3. **OCR表格识别**：使用视觉模型逐行识别表格数据，避免跳行、漏行
4. **交易记录提取**：从OCR结果中提取结构化的交易记录数据
5. **余额自动验证**：根据前后余额验证交易金额的正确性
6. **问题行标黄**：有问题的数据行自动标黄，并在核查备注中说明原因

## 核心原则
1. **忠实原数据**：只提取PDF中明确显示的数据，不推测、补充或修改
2. **保留脱敏信息**：账号中的星号（*）保留原样
3. **逐行识别**：OCR识别时逐行提取，避免跳行或错行
4. **余额验证**：通过前后余额验证交易金额的正确性
5. **问题标记**：验证失败或数据异常的行标黄并备注原因

## 工作流程
1. PDF → 图片（300 DPI，支持多页）
2. 图片 → 银行信息识别（视觉模型）
3. 图片 → OCR表格识别（视觉模型，逐行提取）
4. OCR结果 → 结构化数据（大模型）
5. 数据校验与余额验证（大模型）
6. 生成Excel（问题行标黄）

## 余额验证逻辑
```
验证公式：上条余额 + 本条收入 - 本条支出 = 本条余额
```

### 验证场景
1. **第一条记录**：无法验证，标记"首条记录，初始余额：XXX"
2. **中间记录**：根据上一条余额验证
3. **余额不匹配**：标记"余额验证失败：期望XXX，实际YYY，差额ZZZ"
4. **缺少余额**：标记"缺少余额数据，无法验证"

### 允许误差
- 浮点数精度误差：±0.01

## 问题标黄规则
以下情况的行会被标黄：
1. 余额验证失败
2. 缺少余额数据
3. 金额异常（负数、超大值等）
4. 日期格式异常
5. 关键字段缺失

## 注意事项
1. **不要过度引申**：只提取PDF中明确显示的数据
2. **保留原格式**：日期、账号等保持原PDF中的格式
3. **星号保留**：账号中的星号（*）不要尝试补充或猜测
4. **余额验证**：余额是验证数据正确性的关键字段
5. **OCR优势**：使用视觉模型识别表格，避免跳行和错行问题
